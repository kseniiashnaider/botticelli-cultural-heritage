Methodology

1. Introduction

The purpose of this project is to present the complete methodology used to identify, validate and enrich semantic knowledge gaps in the ArCo Knowledge Graph. Rather than simply listing missing information, this report explains the entire workflow followed during the project. Every proposed enrichment is supported by SPARQL exploration, comparison with the existing RDF description, validation through multiple Large Language Models and transformation into RDF triples. This section represents the final methodological summary of the Knowledge Engineering project.

2. Research Objective

The selected case study is Sandro Botticelli's Madonna con Bambino, San Giovannino e angeli. The main objective was to determine whether the ArCo Knowledge Graph already contained all relevant semantic information describing the artwork or whether important interpretative concepts were absent. If semantic gaps were confirmed, the project aimed to generate validated knowledge suitable for RDF enrichment while preserving the original ArCo metadata.

3. Overall Workflow

  1. Selection of the cultural property.
  2. Exploration of the ArCo Knowledge Graph using SPARQL.
  3. Verification of existing metadata.
  4. Identification of evidence-based semantic gaps.
  5. Extraction of contextual knowledge using ChatGPT and Gemini.
  6. Comparison and validation of generated knowledge.
  7. Transformation of validated concepts into RDF triples.
  8. Proposal for semantic enrichment of the Knowledge Graph.

4. Exploration of Existing Knowledge

The first phase focused on understanding what information was already available in ArCo. SPARQL Query 1 verified that the selected URI represented a valid HistoricOrArtisticProperty. Query 2 retrieved the complete RDF description of the artwork, revealing rich cataloguing metadata including authorship, dating, materials, dimensions, inventory identifiers, bibliography, documentation, descriptions and image references. Query 3 confirmed that the painting already possesses a structured subject through the property hasSubject. These results demonstrated that ArCo already provides reliable administrative and descriptive information. See SPARQL.

9. Semantic Enrichment Strategy

The enrichment strategy deliberately preserved every existing ArCo statement. Only concepts confirmed to be absent from the graph were represented as additional RDF triples. Existing ontology properties were reused whenever possible, while illustrative properties were introduced only for concepts that currently lack direct representation in ArCo. This approach maintains interoperability while demonstrating how knowledge graphs can be extended through carefully validated semantic information. See RDF Enrichment.

13. RDF Enrichment Workflow

  1. Identify a confirmed semantic gap.
  2. Retrieve candidate knowledge using LLMs.
  3. Compare responses across models.
  4. Validate the information using scholarly sources.
  5. Map concepts to existing ArCo properties whenever possible.
  6. Introduce illustrative properties only where ArCo has no equivalent.
  7. Generate RDF triples.
  8. Preserve compatibility with the original Knowledge Graph.
