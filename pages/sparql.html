Sandro Botticelli in the ArCo Knowledge Graph

SPARQL Queries & Results

This section presents the main SPARQL queries used to explore the ArCo knowledge graph and analyse the RDF description of the selected Botticelli painting.

Anchor resource:

https://w3id.org/arco/resource/HistoricOrArtisticProperty/1201008107

Queries 3–6 use this URI through VALUES. Queries 3–6 are also discussed in the Knowledge Gaps section.

ArCo SPARQL Endpoint

Query 1 — Finding Botticelli-related resources

The first step was to confirm that ArCo contains resources related to Botticelli and to select the main artwork for the project.

Explanation of keywords used

  • rdfs:label — textual label of a resource
  • FILTER — restricts results to a condition
  • REGEX — pattern search inside a string (case-insensitive flag "i")
  • DISTINCT — removes duplicate rows
  • ORDER BY — sorts results by label
  • LIMIT — caps the number of rows returned

SPARQL Query

PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT DISTINCT ?cp ?label
WHERE {
  ?cp rdfs:label ?label .
  FILTER(REGEX(STR(?label), "botticelli", "i"))
}
ORDER BY ?label
LIMIT 50

Results

The query confirmed that ArCo contains numerous resources whose labels mention Botticelli, including paintings, photographic reproductions and cultural events. The resource HistoricOrArtisticProperty/1201008107 was selected because it is a catalogued painting with explicit attribution to Sandro Botticelli.

Query 2 — Exploring the direct RDF description

After identifying the anchor resource, we retrieved all properties and values directly connected to it.

Explanation of keywords used

  • VALUES — restricts the query to the selected resource URI
  • ?p / ?o — predicate and object of each triple
  • OPTIONAL — retrieves human-readable labels when available
  • ORDER BY — sorts by predicate

SPARQL Query

PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT DISTINCT ?p ?pLabel ?o ?oLabel
WHERE {
  VALUES ?cp { <https://w3id.org/arco/resource/HistoricOrArtisticProperty/1201008107> }

  ?cp ?p ?o .

  OPTIONAL { ?p rdfs:label ?pLabel . }
  OPTIONAL { ?o rdfs:label ?oLabel . }
}
ORDER BY ?p
LIMIT 100

Results

The query shows that the painting is described through type, labels, location, catalogue links and rdfs:seeAlso connections to photographic documentation. However, the direct RDF description does not explicitly represent philosophical or interpretative context such as Neoplatonism.

Query 3 — Checking structured subject and catalogue relations

This query checks whether contextual information is already modelled through ArCo Context Description or Catalogue modules.

Explanation of keywords used

  • UNION — combines two graph patterns in one result set
  • a-cd:hasSubject — links a cultural property to a subject resource (Context Description)
  • arco-catalogue: / roapit: — catalogue and role-in-time properties
  • OPTIONAL — optional labels for predicates and objects

SPARQL Query

PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>
PREFIX a-cd: <https://w3id.org/arco/ontology/context-description/>
PREFIX arco-catalogue: <https://w3id.org/arco/ontology/catalogue/>
PREFIX roapit: <https://w3id.org/italia/onto/RO/>

SELECT DISTINCT ?relation ?object ?objectLabel
WHERE {
  VALUES ?cp { <https://w3id.org/arco/resource/HistoricOrArtisticProperty/1201008107> }

  {
    ?cp a-cd:hasSubject ?object .
    BIND("hasSubject" AS ?relation)
  }
  UNION
  {
    ?cp arco-catalogue:hasCatalogueRecordVersion/arco-catalogue:hasCatalogueRecordVersionRiT ?rit .
    ?rit roapit:isRoleInTimeOf ?object .
    BIND("author (catalogue)" AS ?relation)
  }

  OPTIONAL { ?object rdfs:label ?objectLabel . }
}
ORDER BY ?relation
LIMIT 50

Query 4 — Structured relations vs photographic labels

This query uses UNION to compare information modelled as direct semantic relations with philosophical and cultural entities mentioned only in labels of photographic resources linked through rdfs:seeAlso.

Explanation of keywords used

  • UNION — compares two types of evidence in one query
  • rdfs:seeAlso — links the main resource to related documentation
  • REGEX — detects named entities inside photographic labels
  • COUNT / GROUP BY — counts resources per entity
  • ORDER BY DESC(?numberOfResources) — ranks by frequency

SPARQL Query

PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>
PREFIX a-cd: <https://w3id.org/arco/ontology/context-description/>

SELECT ?source ?entityName (COUNT(DISTINCT ?item) AS ?numberOfResources)
WHERE {
  VALUES ?cp { <https://w3id.org/arco/resource/HistoricOrArtisticProperty/1201008107> }

  {
    ?cp a-cd:hasSubject ?item .
    BIND("Direct semantic relation" AS ?source)
    OPTIONAL { ?item rdfs:label ?lbl . }
    BIND(COALESCE(?lbl, STR(?item)) AS ?entityName)
  }

  UNION

  {
    ?cp rdfs:seeAlso ?item .
    ?item rdfs:label ?photoLabel .
    BIND("Photographic resource label" AS ?source)

    BIND(
      IF(REGEX(STR(?photoLabel), "Neoplaton", "i"), "Neoplatonism",
      IF(REGEX(STR(?photoLabel), "Ficino", "i"), "Marsilio Ficino",
      IF(REGEX(STR(?photoLabel), "Platon", "i"), "Platonic philosophy",
      IF(REGEX(STR(?photoLabel), "Humanism", "i"), "Renaissance Humanism",
      IF(REGEX(STR(?photoLabel), "Medici", "i"), "Medici circle",
      ""))))) AS ?entityName
    )
    FILTER(?entityName != "")
  }
}
GROUP BY ?source ?entityName
ORDER BY ?source DESC(?numberOfResources)

Interpretation

If philosophical entities appear mainly under photographic labels but not as direct a-cd:hasSubject relations, the information exists in ArCo but is not represented at the appropriate semantic level — this supports Gap 1.

This query counts how often selected philosophical and cultural entities appear in labels of resources linked to the anchor painting.

SPARQL Query

PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

Query 6 — Checking direct links to philosophical entities

This query checks whether philosophical entities are already directly linked to the anchor resource through structured properties other than rdfs:seeAlso.

Explanation of keywords used

  • FILTER(?p != rdfs:seeAlso) — excludes photographic links
  • REGEX — searches object values and labels for selected names
  • OPTIONAL — retrieves labels for properties and objects

SPARQL Query

PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT DISTINCT ?p ?pLabel ?o ?oLabel
WHERE {
  VALUES ?cp { <https://w3id.org/arco/resource/HistoricOrArtisticProperty/1201008107> }

  ?cp ?p ?o .

  OPTIONAL { ?p rdfs:label ?pLabel . }
  OPTIONAL { ?o rdfs:label ?oLabel . }

  FILTER(?p != rdfs:seeAlso)

  FILTER(
    REGEX(STR(?o), "Neoplaton|Ficino|Platon|Humanism|Medici", "i") ||
    REGEX(STR(?oLabel), "Neoplaton|Ficino|Platon|Humanism|Medici", "i")
  )
}
ORDER BY ?p
LIMIT 100

Interpretation

An empty result set confirms that the selected philosophical entities are not directly connected to the painting through structured RDF properties. They appear mainly through related photographic documentation. This is the basis for Gap 2 and the proposed enrichment in RDF Enrichment.

Conclusion of the SPARQL exploration

The SPARQL exploration shows that Botticelli's painting is documented in ArCo with catalogue and photographic metadata. Important interpretative information, however, is often present only indirectly in textual labels rather than as explicit semantic relations of the main resource. These findings are developed in the Knowledge Gaps section.
